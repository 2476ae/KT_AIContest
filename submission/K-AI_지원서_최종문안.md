### 문제 정의 및 해결 필요성

이 프로젝트는 월말이 되어서야 예산 초과를 알아차리는 문제에서 출발했습니다. 식비나 카페, 교통비처럼 자주 발생하는 지출은 한 건씩 보면 부담이 작아 보이지만, 여러 날짜와 결제 수단에 흩어지면 전체 흐름을 파악하기 어렵습니다. 매달 자동으로 빠져나가는 구독료도 금액이 작다는 이유로 점검 대상에서 쉽게 빠집니다.

기존 가계부는 지출을 기록하고 통계를 보여주는 데 강점이 있지만, 사용자가 그 결과를 직접 해석해야 하는 경우가 많습니다. 금융 관리에 익숙하지 않은 대학생이나 사회초년생에게는 “이번 달에 얼마나 더 써도 되는지”, “어느 지출부터 살펴봐야 하는지”를 판단하는 일 자체가 부담이 됩니다. 머니루틴 캘린더는 기록에서 끝나지 않고, 목표 소비액과 저축액을 기준으로 다음 행동을 정할 수 있게 돕고자 만들었습니다.

### 대상

주요 사용자는 생활비와 구독료를 스스로 관리하기 시작한 대학생, 취업 준비생, 사회초년생입니다. 이들은 수입이 크지 않거나 매달 사용할 수 있는 금액이 일정하지 않아 작은 지출 변화에도 월말 예산이 크게 흔들릴 수 있습니다. 사용자는 앱에서 소비 흐름을 직접 확인하고 조정할 수 있으며, 장기적으로는 대학의 학생지원센터나 청년지원기관에서도 금융 습관 교육을 위한 체험 도구로 활용할 수 있습니다.

## 3. 프로그램 및 AI 활용 설명

### 프로그램 설명

처음 접속한 사용자는 금융기관 인증 없이 샘플 데이터로 서비스를 바로 체험할 수 있습니다. 자신의 거래를 관리하고 싶다면 금액과 사용처를 직접 입력하거나 CSV 파일을 불러오면 됩니다. 입력한 소비는 월간 캘린더에 날짜별로 표시되고, 과소비가 발생한 날과 정기결제일을 구분해서 볼 수 있습니다.

목표 화면에서는 월수입, 목표 소비액, 목표 저축액과 집중해서 살펴볼 소비 분야를 설정합니다. 앱은 현재 지출과 남은 날짜를 바탕으로 오늘 사용할 수 있는 금액을 계산합니다. AI 코치 화면에서는 이 계산 결과를 근거로 오늘의 소비 가이드, 이번 주 미션, 정기결제 점검 항목을 제시합니다. 소비를 무조건 줄이라고 요구하기보다, 목표 범위 안에 있다면 현재 계획을 유지하도록 안내하고 조정이 필요할 때만 구체적인 행동을 제안하도록 설계했습니다.

모든 거래와 목표 정보는 기본적으로 사용자의 브라우저에 저장됩니다. 설정 화면에서 CSV로 내보내거나 전체 데이터를 초기화할 수 있으며, 실제 계좌번호나 카드번호, 금융 인증정보는 받지 않습니다.

### 개발 과정

먼저 공모전의 경제·금융 분야 예시와 심사 기준을 분석해 “작은 소비와 구독료를 월말 전에 알아차리기 어렵다”는 문제를 핵심 과제로 정했습니다. 이후 일반적인 통계형 가계부와 차이를 만들기 위해 소비 달력, 목표 설정, 행동 중심 AI 코치를 하나의 사용 흐름으로 연결했습니다.

기능 구현은 데이터 구조와 계산 규칙을 먼저 정한 뒤 화면을 얹는 순서로 진행했습니다. 거래 추가·수정·삭제, CSV 가져오기, 목표 계산, 반복결제 탐지를 TypeScript 로직으로 구현하고, 같은 값이 홈·캘린더·목표·AI 코치 화면에 일관되게 반영되는지 단위 테스트로 확인했습니다. 그다음 모바일 금융 서비스에 익숙한 사용자가 설명서 없이도 이용할 수 있도록 하단 내비게이션과 카드형 화면을 구성했습니다.

AI 기능은 계산 결과를 대신 결정하지 않도록 경계를 두었습니다. 예산과 저축 전망은 검증 가능한 로컬 계산으로 확정하고, OpenAI API에는 거래 분류나 행동 문구 보완에 필요한 최소 정보만 전달합니다. 외부 AI가 늦거나 응답하지 않을 때도 기본 분석으로 전환되어 앱 사용이 중단되지 않게 했습니다.

마지막 단계에서는 실제 심사 흐름을 기준으로 자동화 테스트를 작성했습니다. 샘플 데이터 불러오기, 목표 변경, 거래 추가, 캘린더 반영, AI 응답 실패 시 복구, 모바일 키보드와 하단 메뉴 간섭 등을 데스크톱과 모바일 환경에서 반복 검증했습니다. 완성된 앱은 GitHub와 Vercel을 연결해 배포하고, GitHub Pages를 예비 접속 경로로 함께 운영하고 있습니다.

### 개발 과정에서 사용한 핵심 지시문 요약

아래 문장은 개발 과정에서 Codex에 전달한 요구사항을 신청서용으로 정리한 것입니다. 세부 수정 지시는 많았지만, 설계 결정에 영향을 준 내용은 개발 순서대로 남겼습니다.

```text
1. K-AI 콘텐츠 공모전 Track B의 경제·금융 분야에 맞춰, 대학생과 사회초년생이 반복 소비와 구독료를 쉽게 파악할 수 있는 웹앱을 설계해줘. 단순한 통계 화면이 아니라 사용자가 다음 행동을 정할 수 있어야 해.

2. 금융 인증 없이도 심사위원이 바로 체험할 수 있도록 샘플 데이터, 직접 입력, CSV 가져오기를 제공하고, 거래는 소비 캘린더에서 날짜별로 확인할 수 있게 해줘.

3. 목표 소비액과 목표 저축액을 기준으로 오늘 사용 가능한 금액을 계산해줘. 숫자는 TypeScript 로직에서 먼저 검증하고, 생성형 AI가 예산 금액이나 상태를 임의로 바꾸지 못하게 해줘.

4. AI 코치는 사용자를 탓하거나 무조건 절약을 강요하지 않아야 해. 목표 범위 안이면 현재 계획을 유지하도록 안내하고, 조정이 필요할 때만 금액과 행동이 분명한 미션을 제안해줘.

5. OpenAI API 키는 브라우저에 노출하지 말고 서버 프록시에서만 사용해줘. 응답 지연, 호출 실패, 사용량 제한 상황에서는 로컬 분석으로 전환하고 화면이 멈추지 않게 해줘.

6. 390px 모바일 화면과 데스크톱에서 같은 핵심 흐름을 검증하고, 입력 중 키보드가 저장 버튼이나 하단 메뉴를 가리지 않도록 해줘. 샘플 로드부터 목표 변경, 거래 추가, AI 코치 확인까지 Playwright 테스트로 자동화해줘.
```

### AI 활용 설명

개발 과정에서는 OpenAI Codex를 활용해 요구사항을 구조화하고, React·TypeScript 코드 작성과 리팩터링, 테스트 케이스 설계, 오류 원인 점검을 진행했습니다. 기능을 한 번에 생성해 그대로 사용하는 방식이 아니라, 화면별 요구사항과 계산 기준을 나누어 전달하고 실제 실행 결과를 확인하면서 수정했습니다.

서비스 안에서는 OpenAI Responses API를 두 곳에 사용합니다. 사용처명과 메모를 바탕으로 거래 카테고리를 분류하고, 앱이 먼저 계산한 소비 상태와 미션 문구를 사용자가 읽기 편한 표현으로 다듬습니다. 예산 진행률, 오늘 한도, 저축 전망, 반복결제 후보는 TypeScript 규칙으로 먼저 계산되며 AI가 해당 숫자를 변경할 수 없습니다. 이렇게 계산과 문장 생성을 분리해 금융 정보가 그럴듯한 문장 때문에 왜곡되는 위험을 줄였습니다.

AI 응답은 정해진 JSON 구조로만 받도록 제한하고, 8초 안에 응답하지 않거나 형식이 맞지 않으면 로컬 분석 결과를 그대로 보여줍니다. API 키는 Vercel 서버 함수에만 보관하며 브라우저 코드에는 포함하지 않았습니다. 호출 횟수 제한과 중복 요청 방지, 결과 캐시도 적용해 공개된 심사 URL에서 불필요한 비용이 발생하지 않도록 했습니다.

### 서비스에서 사용하는 핵심 AI 프롬프트

거래 분류 프롬프트:

```text
너는 한국어 소비 내역을 분류하는 금융 코칭 보조 AI다.
반드시 허용된 카테고리 중 하나만 선택한다.
정기 결제, OTT, 멤버십, 클라우드, 앱스토어형 사용처는 구독으로 분류한다.
사용자가 읽기 쉬운 한국어 한 문장으로 분류 이유를 작성한다.

입력: 사용처명, 메모, 정기결제 여부
출력: 식비, 카페/간식, 교통, 쇼핑, 여가, 구독, 교육, 의료, 생활, 기타 중 하나와 짧은 이유
```

소비 코칭 문구 보완 프롬프트:

```text
너는 대학생과 사회초년생을 위한 목표 기반 소비 코치다.

카테고리별 행동과 미션 문구만 더 친절하고 구체적으로 다듬는다.

사용자를 탓하지 않고 짧은 완결 문장으로 안내한다.
투자 권유나 계좌 연결 요청은 하지 않는다.
```

### 활용 데이터

제출 버전은 프로젝트에서 직접 만든 가상 소비 데이터와 사용자가 입력한 거래를 활용합니다. 데이터 항목은 날짜, 사용처, 금액, 메모, 소비 카테고리, 정기결제 여부이며, 은행 계좌나 카드사에서 개인정보를 수집하지 않습니다. CSV 파일도 사용자가 직접 불러오는 경우에만 브라우저에서 처리합니다.

거래 분류를 요청할 때는 사용처명과 메모, 정기결제 여부만 외부 AI에 전달합니다. 코칭 문구를 보완할 때는 전체 거래 원문 대신 앱이 계산한 월간 요약, 카테고리별 상태, 미션 초안만 전달합니다. 데이터는 소비 유형을 분류하고 목표 예산에 맞는 행동 안내를 만드는 목적으로만 사용하며, 애플리케이션 서버나 별도 데이터베이스에 저장하지 않습니다.

### 차별성

머니루틴 캘린더의 중심 화면은 통계 차트가 아니라 달력입니다. 사용자는 날짜별 소비 상태와 정기결제일을 함께 보면서 “언제 지출이 몰렸는지”를 바로 확인할 수 있습니다. 여기에 목표 소비액과 저축액을 연결해, 이미 쓴 돈을 설명하는 데서 끝나지 않고 오늘의 한도와 다음 행동까지 제안합니다.

또 다른 차이는 AI가 모든 판단을 맡지 않는다는 점입니다. 금융 숫자와 상태는 테스트 가능한 규칙으로 계산하고, AI는 분류와 문장 보완처럼 강점이 있는 역할에만 사용했습니다. 외부 AI가 실패해도 같은 계산 결과를 계속 확인할 수 있어 심사 환경이나 실제 사용 중 네트워크 상태에 덜 영향을 받습니다. 실제 금융 인증을 요구하지 않는 체험 구조와 사용자가 직접 데이터를 내보내고 지울 수 있는 점도 초기 금융 서비스에서 필요한 신뢰를 고려한 설계입니다.

## 4. 기대 효과

### 기대 효과

사용자는 월말 결산을 기다리지 않고 현재 소비 속도와 남은 예산을 확인할 수 있습니다. 특히 한 번에 큰돈을 쓰는 경우보다 매일 조금씩 반복되는 식비, 카페, 구독료가 예산에 어떤 영향을 주는지 알아보기 쉬워집니다. 숫자만 보여주는 대신 오늘 할 수 있는 행동을 함께 제시하므로 금융 관리 경험이 적은 사람도 자신의 소비를 스스로 조정하는 연습을 할 수 있습니다.

서비스가 반복적으로 사용되면 사용자는 지출을 무조건 참는 대신 목표 안에서 계획하는 습관을 만들 수 있습니다. 이는 청년층의 금융 문해력을 높이고, 작은 예산 초과가 누적되어 생활비 부족으로 이어지는 상황을 줄이는 데 도움이 될 것으로 기대합니다.

### 향후 계획

고도화 단계에서는 사용자가 명시적으로 동의한 경우에만 오픈뱅킹이나 금융 마이데이터를 연결해 거래 입력 과정을 줄일 계획입니다. 자동 연동 전에는 데이터 저장 기간, 삭제 방법, AI에 전달되는 범위를 사용자가 직접 선택할 수 있도록 설계하겠습니다.

기능 측면에서는 반복결제 알림, 월별 소비 변화 비교, 개인별 목표 달성 패턴에 맞춘 미션, 가족이나 금융 멘토와 목표만 선택적으로 공유하는 기능을 검토하고 있습니다. 대학 학생지원센터나 청년지원기관과 협력할 수 있다면 실제 사용성 검증을 거쳐 금융 습관 교육용 버전으로 확장하고자 합니다.

## 5. 팀 소개

### 구성 및 역할 - 개인 참가 시

1인 개발로 기획, 화면 설계, 프론트엔드 구현, AI 서버 연동, 테스트 작성, 배포와 문서화를 직접 진행했습니다.

### 구성 및 역할 - 팀 참가 시

아래 형식으로 실제 역할만 작성합니다.

```text
[이름 1] 프로젝트 기획 및 서비스 정책, 사용자 흐름 설계
[이름 2] React·TypeScript 프론트엔드와 소비 분석 로직 구현
[이름 3] OpenAI 서버 연동, 테스트 자동화와 배포
```

### 실행 역량

아이디어 제안에 머물지 않고 심사위원이 바로 사용할 수 있는 웹앱으로 완성했습니다. React와 TypeScript로 화면과 데이터 흐름을 구현했고, Vercel 서버 함수로 OpenAI API 키를 브라우저와 분리했습니다. 현재 버전은 Vitest 13개 파일의 단위 테스트 68개와 Playwright 데스크톱·모바일 시나리오 24개를 통과했습니다. Vercel 운영 URL과 GitHub Pages 예비 URL을 함께 준비해 접속 장애에도 대응할 수 있도록 했습니다.

### 주요 실적

관련 수상이나 사업 실적이 없다면 `해당 없음`으로 작성합니다. 증빙할 수 없는 성과는 기재하지 않습니다.

## 6. 입력란이 짧을 때 사용할 축약문

### 문제 정의 축약문

대학생과 사회초년생은 작은 지출과 구독료가 여러 날짜에 흩어져 월말이 되어서야 예산 초과를 알아차리는 경우가 많습니다. 기존 가계부는 기록과 통계를 제공하지만, 사용자가 어디부터 조정해야 할지 직접 해석해야 합니다. 머니루틴 캘린더는 목표 소비액과 저축액을 기준으로 현재 소비 속도와 오늘의 한도를 계산하고, 다음 행동까지 안내해 이 문제를 해결합니다.

### 프로그램 설명 축약문

사용자는 샘플 데이터, 직접 입력, CSV 가져오기로 금융 인증 없이 서비스를 체험할 수 있습니다. 거래는 월간 달력에 표시되며 과소비일과 정기결제일을 구분해 볼 수 있습니다. 목표 소비액과 저축액을 설정하면 오늘 한도와 저축 전망을 계산하고, AI 코치가 소비 가이드와 실천 미션을 제시합니다. 실제 계좌·카드 인증정보는 수집하지 않습니다.

### AI 활용 축약문

OpenAI Responses API는 거래 카테고리 분류와 행동 문구 보완에 사용합니다. 예산 진행률, 오늘 한도, 저축 전망과 반복결제 후보는 TypeScript 규칙으로 먼저 계산하며 AI가 숫자나 상태를 바꿀 수 없습니다. 응답이 늦거나 실패하면 로컬 분석으로 전환하고, API 키는 서버 함수에만 보관합니다.

### 차별성 축약문

머니루틴 캘린더는 소비를 차트가 아닌 달력으로 보여주고, 목표 저축과 연결해 오늘의 행동까지 제안합니다. 금융 숫자는 검증 가능한 규칙으로 계산하고 AI는 분류와 문장 보완에만 사용해 신뢰성을 높였습니다. 금융 인증 없이 체험할 수 있고 사용자가 데이터를 직접 내보내거나 삭제할 수 있습니다.
