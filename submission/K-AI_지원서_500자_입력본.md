# K-AI 콘텐츠 공모전 500자 제한 입력본

온라인 신청서에 붙여 넣기 위한 최종 문안이다. 각 답변은 코드 블록 안의 본문만 복사한다. 글자 수는 공백과 문장부호를 포함해 500자 미만으로 확인한다.

## 솔루션명

```text
머니루틴 캘린더
```

## 한 줄 소개

```text
소비 흐름을 달력으로 보여주고, 목표 예산에 맞춰 오늘의 행동을 안내하는 청년 맞춤형 AI 소비 코치입니다.
```

## 문제 정의 및 해결 필요성

```text
가계부를 쓰다 보면 한 번의 큰 지출보다 식비, 카페, 교통비처럼 매일 반복되는 작은 소비를 놓치기 쉽습니다. 구독료도 금액이 작아 계속 결제되는 경우가 많고, 결국 월말이 되어서야 예산을 넘겼다는 사실을 알게 됩니다. 기존 서비스는 지출 통계를 잘 보여주지만, 금융 관리에 익숙하지 않은 사용자는 그 숫자를 보고 무엇부터 바꿔야 할지 판단하기 어렵습니다. 머니루틴 캘린더는 소비 기록을 목표 소비액과 저축액에 연결해 현재 속도를 보여주고, 다음 소비 전에 할 수 있는 행동을 안내하기 위해 만들었습니다.
```

## 대상

```text
주요 사용자는 생활비를 직접 관리하기 시작한 대학생, 취업 준비생, 사회초년생입니다. 이 시기에는 수입이 많지 않거나 매달 쓸 수 있는 금액이 일정하지 않아 작은 지출 변화에도 월말 생활비가 크게 흔들릴 수 있습니다. 앱은 복잡한 금융 용어 대신 오늘 쓸 수 있는 금액과 점검할 소비를 쉬운 문장으로 보여줍니다. 향후에는 대학 학생지원센터나 청년지원기관의 금융 습관 교육에도 활용할 수 있습니다.
```

## 프로그램 설명

```text
처음 접속하면 금융기관 인증 없이 샘플 데이터로 서비스를 바로 체험할 수 있습니다. 자신의 소비는 직접 입력하거나 CSV 파일로 불러옵니다. 거래가 등록되면 월간 달력에서 날짜별 지출, 과소비일, 정기결제일을 확인할 수 있습니다. 목표 화면에서 월수입과 목표 소비액·저축액을 설정하면 현재 지출과 남은 날짜를 기준으로 오늘 한도가 계산됩니다. AI 코치는 이 결과를 바탕으로 오늘의 소비 가이드와 실천 미션을 보여줍니다. 데이터는 브라우저에 저장되며 사용자가 직접 내보내거나 초기화할 수 있습니다.
```

## 개발 과정

```text
공모전의 경제·금융 분야와 심사 기준을 살펴본 뒤, 작은 소비와 구독료를 월말 전에 알아차리기 어렵다는 문제를 핵심 과제로 정했습니다. 화면부터 만들기보다 거래 구조, 목표 계산, 반복결제 판단 규칙을 먼저 구현했고, 같은 결과가 홈·캘린더·목표 화면에 일관되게 표시되는지 테스트했습니다. 이후 달력을 중심으로 모바일 화면을 구성하고 OpenAI 연동을 추가했습니다. 금융 숫자는 로컬 코드가 계산하고 AI는 분류와 문장 보완만 맡도록 경계를 뒀습니다. 마지막으로 데스크톱과 모바일 핵심 흐름을 자동화 테스트로 확인한 뒤 Vercel에 배포했습니다.
```

## AI 활용 설명

```text
개발 과정에서는 OpenAI Codex를 이용해 요구사항 정리, React·TypeScript 코드 작성, 리팩터링과 테스트 설계를 진행했습니다. 서비스에서는 OpenAI Responses API를 거래 카테고리 분류와 행동 문구 보완에 사용합니다. 예산 진행률, 오늘 한도, 저축 전망과 반복결제 후보는 TypeScript 규칙으로 먼저 계산하며 AI가 이 숫자를 바꿀 수 없습니다. API 키는 Vercel 서버 함수에만 보관합니다. 응답이 8초를 넘거나 형식이 맞지 않으면 로컬 분석으로 전환해 서비스 이용이 중단되지 않도록 했습니다.
```

## 핵심 AI 프롬프트

```text
너는 대학생과 사회초년생을 위한 목표 기반 소비 코치다. 앱이 계산한 baseReport를 최종 결과로 보고 숫자, 상태, 판단 방향을 바꾸지 않는다. 카테고리별 행동과 미션 문구만 친절하고 구체적으로 다듬는다. 금액이 계획 안이면 초과나 절감 필요라고 쓰지 않는다. 사용자를 탓하지 않고 짧은 완결 문장으로 안내한다. 투자 권유나 계좌 연결을 요청하지 않는다. 거래 분류 시에는 허용된 카테고리 중 하나만 선택하고, 사용처와 메모를 근거로 사용자가 이해하기 쉬운 분류 이유를 한 문장으로 작성한다.
```

## 활용 데이터

```text
프로젝트에서 직접 만든 가상 소비 데이터와 사용자가 입력한 거래를 활용합니다. 항목은 날짜, 사용처, 금액, 메모, 소비 카테고리, 정기결제 여부이며 실제 계좌번호나 카드 인증정보는 받지 않습니다. 거래는 기본적으로 사용자의 브라우저에 저장됩니다. 외부 AI에는 분류에 필요한 사용처와 메모, 정기결제 여부 또는 앱이 계산한 월간 요약만 전달합니다. 원본 거래 전체를 별도 서버나 데이터베이스에 보관하지 않습니다.
```

## 차별성

```text
머니루틴 캘린더는 소비를 통계 차트보다 달력에서 먼저 보여줍니다. 사용자는 언제 지출이 몰렸고 정기결제가 언제 나가는지 날짜 흐름으로 확인할 수 있습니다. 이미 쓴 돈을 정리하는 데서 끝나지 않고 목표 소비액과 저축액을 연결해 오늘의 한도와 다음 행동까지 제안합니다. 또 금융 숫자는 검증 가능한 로컬 규칙으로 계산하고 AI는 분류와 문장 보완에만 사용했습니다. 외부 AI가 실패해도 기본 분석이 유지되며, 금융 인증 없이 체험하고 데이터를 직접 삭제할 수 있습니다.
```

## 기대 효과

```text
사용자는 월말 결산을 기다리지 않고 현재 소비 속도와 남은 예산을 확인할 수 있습니다. 특히 식비, 카페, 구독료처럼 조금씩 반복되는 지출이 생활비에 어떤 영향을 주는지 알아보기 쉬워집니다. 숫자와 함께 바로 실행할 수 있는 행동을 제시하므로 금융 관리 경험이 적은 사람도 스스로 소비를 조정하는 연습을 할 수 있습니다. 장기적으로는 무조건 아끼기보다 목표 안에서 계획해 쓰는 습관과 청년층의 금융 문해력 향상에 도움이 될 것으로 기대합니다.
```
