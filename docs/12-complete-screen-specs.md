# v4 전체 화면 상세 명세

## 목적

이 문서는 머니루틴 캘린더의 전체 화면을 v4 디자인 방향에 맞춰 구현하기 위한 화면별 명세다. 첫 제출작은 모든 기능을 깊게 구현하기보다, 모바일에서 실제 배포 가능한 개인 금융 코치 앱처럼 보이는 완성형 경험을 목표로 한다.

대표 참고 이미지는 `mockups/mobile-v4-reference-synthesis.png`다. 이 이미지는 지향할 첫인상과 정보 위계를 보여주는 자료이며, 실제 구현에서 동일하게 복제할 필요는 없다.

## 전체 화면 목록

| 화면 | 역할 | 심사에서 보여줄 가치 |
| --- | --- | --- |
| 홈 | 현재 소비 상태, 목표 진행률, 소비 캘린더, 코치 미션 요약 | 첫 10초 안에 앱 완성도와 목적 전달 |
| 캘린더 | 날짜별 소비 흐름과 구독 결제일 탐색 | 이 앱만의 차별화된 UX |
| 추가/입력 | 직접 입력, CSV 연결, 분류 수정 | 실제로 쓸 수 있는 웹앱이라는 증거 |
| 목표 | 목표 소비액과 저축액 조정 | AI 코칭이 목표 기반이라는 점 |
| 코치 | 오늘의 조정, 미션, 구독 점검 | AI가 행동으로 연결된다는 점 |
| 설정 | 샘플 데이터, 개인정보 안내, 알림 설정 | 신뢰와 데모 안정성 |

## 공통 앱 구조

### 모바일 우선

- 앱 폭은 430px 기준으로 설계한다.
- 배경은 `#F5F7FA`, 카드는 흰색을 사용한다.
- 하단 탭은 `홈`, `캘린더`, `추가`, `코치`, `설정`으로 고정한다.
- `추가` 탭만 파란색으로 강하게 강조한다.
- 하단 탭이 콘텐츠를 가리지 않도록 하단 패딩을 둔다.

### 데스크톱 확장

데스크톱은 모바일 화면을 그대로 늘리지 않는다. 모바일 홈의 정보 순서를 유지하되, 캘린더와 보조 카드를 2열로 확장한다.

기본 구조:

```text
좌측: HeroSpendingCard, QuickActionGrid, SpendingCalendar
우측: CoachMission, SubscriptionInsight, RecentTransactions
```

## 1. 홈

### 화면 목표

이번 달 소비 상태와 오늘 조정할 행동을 바로 보여준다.

### 모바일 구성

1. TopBar: 브랜드, 월 선택, 알림
2. HeroSpendingCard: 이번 달 소비 금액, 목표 진행률, 오늘 가능예산, 저축 예상, 고정 구독비
3. CoachStrip: 오늘의 조정 문장
4. QuickActionGrid: 내역 추가, 파일 연결, 목표 수정, 코치 보기
5. SpendingCalendar: 밝은 소비 캘린더
6. SelectedDayNote: 선택일 해석
7. MissionList: 이번 주 조정 미션 2개
8. MiniInsightCard: 구독 지출, 잔액 흐름
9. TransactionList: 최근 소비 2건

### 우선 정보

- 이번 달 총 소비
- 월 목표 소비액
- 목표 진행률
- 오늘 가능예산
- 목표 저축 예상
- 고정 구독비
- 오늘의 조정

### 성공 기준

- 첫 화면이 소개 페이지처럼 보이지 않는다.
- 캘린더가 홈의 중심 기능처럼 보인다.
- 구독 지출이 첫 화면 안에 보인다.
- AI 기능이 긴 분석문이 아니라 조정 문장과 미션으로 보인다.

## 2. 캘린더

### 화면 목표

사용자가 날짜 기준으로 "언제 돈이 많이 새는지"를 이해하게 한다.

### 모바일 구성

1. 상단 월 이동
2. 상태 요약: 안정일, 과소비일, 구독일
3. 월간 캘린더
4. 필터 칩: 전체, 과소비, 구독, 무지출
5. 선택 날짜 상세
6. 선택 날짜 코치 코멘트
7. 선택 날짜 거래 목록

### 날짜 셀 상태

| 상태 | 의미 | 색 |
| --- | --- | --- |
| 기본 | 일반 소비 | 연한 회색 |
| 안정 | 무지출 또는 낮은 소비 | 민트 |
| 구독 | 구독/고정비 결제 | 앰버 |
| 초과 | 하루 예산 초과 | 코랄 |
| 선택 | 현재 선택 날짜 | 파란색 |

### 선택 날짜 상세

표시 항목:

- 날짜
- 하루 총 지출
- 가장 큰 카테고리
- 거래 목록
- 코치 한 줄

예시:

> 14일은 평소보다 카페 지출이 큰 날

### 성공 기준

- 모바일에서 날짜, 금액, 상태 점이 겹치지 않는다.
- 캘린더 셀 안에는 짧은 금액 또는 짧은 라벨만 들어간다.
- 선택 날짜 설명은 셀 밖의 상세 패널에 표시된다.

## 3. 추가/입력

### 화면 목표

사용자가 소비 데이터를 직접 넣고 CSV로 연결할 수 있게 한다.

### 구성

1. 금액 입력
2. 사용처 입력
3. 날짜 선택
4. 카테고리 선택
5. 구독 여부
6. 저장 버튼
7. CSV 연결 카드

### 직접 입력 규칙

- 금액 입력을 가장 크게 둔다.
- 사용처와 날짜는 한 흐름으로 이어지게 배치한다.
- 카테고리는 pill 또는 segmented control로 보여준다.
- 저장 버튼은 primary blue로 하나만 강하게 둔다.

### CSV 연결 규칙

- 파일 연결은 큰 기술 기능처럼 보이지 않고 금융앱의 보조 액션처럼 보여야 한다.
- 연결 후에는 불러온 거래 수, 자동 분류 수, 구독 후보 수를 짧게 보여준다.

### 성공 기준

- 직접 입력과 CSV 연결이 모두 가능해 보인다.
- 입력 화면이 회계 프로그램처럼 딱딱하지 않다.
- 저장 후 홈 카드와 캘린더가 갱신될 것처럼 보인다.

## 4. 목표

### 화면 목표

사용자가 목표 소비액과 저축액을 조정하면 홈의 숫자와 코치 미션이 바뀐다는 점을 보여준다.

### 필수 입력

- 월 목표 소비액
- 목표 저축액
- 집중 관리 카테고리
- 구독 지출 상한

### 미리보기

목표 변경 시 아래 값을 즉시 보여준다.

- 오늘 가능예산
- 목표 진행률
- 저축 예상
- 코치 문장 변화

### 성공 기준

- 목표 설정이 단순 폼이 아니라 코치의 기준처럼 보인다.
- 무리한 목표에는 경고보다 조정 제안을 먼저 보여준다.
- 홈의 목표 chip과 같은 시각 언어를 쓴다.

## 5. 코치

### 화면 목표

AI가 소비를 분석해 다음 행동을 제안한다는 점을 가장 분명히 보여준다.

### 구성

1. 오늘의 조정
2. 이번 주 미션
3. 구독 점검
4. 카테고리별 코멘트
5. 다음 결제일 알림
6. 분석 기준

### 미션 카드

표시 항목:

- 행동
- 이유
- 예상 절약액
- 완료 상태

예시:

```text
목요일 무지출 하루 만들기
교통비 제외 자동 계산
+9천
```

### 분석 기준

AI 결과가 허공에서 나온 것처럼 보이지 않도록 판단 근거를 카드로 보여준다.

예시:

- 예산 위치: 55% 사용
- 오늘 한도: 322,210원
- 저축 전망: 902,210원
- 지난달 패턴: 식비 +4%p
- 정기 결제: 63,690원

### 성공 기준

- AI 채팅방처럼 보이지 않는다.
- 금액과 행동이 구체적이다.
- 사용자를 비난하지 않는다.
- 투자, 대출, 보험 추천으로 흐르지 않는다.

## 6. 설정

### 화면 목표

데모와 실제 서비스 신뢰를 위한 보조 설정을 제공한다.

### 구성

- 샘플 데이터 초기화
- 개인정보/금융 인증정보 미수집 안내
- 알림 설정
- 목표 기본값
- 데이터 내보내기

### 성공 기준

- 공모전 데모에서 샘플 데이터를 쉽게 초기화할 수 있다.
- 실제 금융 연동이 아직 없다는 점을 자연스럽게 설명한다.
- 설정 화면도 v4의 밝은 카드 톤을 유지한다.

## 공통 빈 상태

### 데이터 없음

문구:

```text
샘플 소비로 바로 보기
금융 인증 없이 예시 데이터로 체험할 수 있어요.
```

디자인:

- 큰 빈 박스를 만들지 않는다.
- 홈의 HeroSpendingCard 안에서 자연스럽게 시작 버튼을 제공한다.

### CSV 없음

문구:

```text
CSV 파일을 연결하면 소비 캘린더가 자동으로 채워져요.
```

### AI 결과 없음

문구:

```text
소비 데이터가 들어오면 오늘의 조정을 계산할게요.
```

## 공통 성공 기준

- 모든 화면은 밝은 금융앱 톤을 유지한다.
- 캘린더와 코치가 같은 제품 안에 있는 것처럼 보여야 한다.
- "AI 리포트"라는 기능명보다 "코치", "조정", "미션"이라는 경험명으로 표현한다.
- 기능이 늘어나도 홈 화면의 정보 순서가 무너지지 않는다.
