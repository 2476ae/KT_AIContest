# v4 기능 구현용 디자인 적용 가이드

## 목적

이 문서는 기능 개발 단계에서 "기능은 붙었는데 디자인이 다시 MVP처럼 보이는 문제"를 막기 위한 적용 가이드다. 기능을 추가할 때마다 아래 매핑을 기준으로 어떤 컴포넌트에 어떤 데이터를 넣을지 결정한다.

## 기본 원칙

### 기능보다 먼저 화면 리듬을 지킨다

새 기능을 추가해도 홈 화면의 리듬은 유지한다.

```text
큰 소비 카드 -> 빠른 액션 -> 소비 캘린더 -> 조정 미션 -> 보조 정보 -> 최근 내역
```

새 기능이 생기면 별도 대시보드 카드를 무작정 추가하지 말고, 위 흐름 중 어디에 들어갈지 먼저 결정한다.

### AI는 기능명이 아니라 결과로 보인다

화면에 `AI 분석`, `AI 리포트`, `AI가 추천` 같은 제목을 반복하지 않는다.

대신:

- 오늘의 조정
- 이번 주 미션
- 구독 점검
- 목표 도달 가능성
- 다음 결제 알림

처럼 사용자가 바로 이해하는 이름을 쓴다.

### 금융앱처럼 신뢰감 있게, 캘린더앱처럼 읽기 쉽게

금융 정보는 숫자가 먼저 보여야 하고, 캘린더 정보는 날짜 흐름이 먼저 보여야 한다. 설명문은 항상 두 번째다.

## 기능별 UI 매핑

### 1. 샘플 데이터 불러오기

표시 위치:

- 첫 방문 상태: HeroSpendingCard 안의 빈 상태 또는 QuickActionGrid의 `파일 연결`
- 불러온 후: 카드와 캘린더가 즉시 채워짐

디자인:

- 빈 상태에서도 큰 빈 박스를 만들지 않는다.
- `샘플 소비로 바로 보기` 버튼 하나를 primary로 둔다.
- 불러온 뒤에는 토스트보다 카드 수치 변화로 피드백을 준다.

카피:

```text
샘플 소비로 바로 보기
금융 인증 없이 예시 데이터로 체험할 수 있어요.
```

### 2. 소비 내역 직접 추가

표시 위치:

- QuickActionGrid의 `내역 추가`
- 하단 탭 가운데 `추가`

화면 구조:

1. 금액
2. 사용처
3. 날짜
4. 카테고리
5. 구독 여부
6. 저장

디자인:

- 금액 입력은 가장 크게 둔다.
- 입력 화면은 카드 여러 개보다 한 흐름의 폼으로 구성한다.
- 저장 버튼은 하단 고정 또는 폼 마지막에 primary로 둔다.

### 3. CSV 파일 연결

표시 위치:

- QuickActionGrid의 `파일 연결`
- 입력 화면 하단의 보조 카드

디자인:

- 파일 업로드는 큰 드롭존보다 작은 금융앱 액션 카드처럼 둔다.
- 업로드 후에는 `35건 불러옴`, `구독 후보 4건 발견`처럼 결과를 짧게 보여준다.

### 4. 목표 설정

표시 위치:

- QuickActionGrid의 `목표 수정`
- HeroSpendingCard의 목표 chip

필드:

- 월 목표 소비액
- 목표 저축액
- 집중 카테고리
- 구독 지출 상한

디자인:

- 숫자는 stepper와 직접 입력을 함께 제공한다.
- 변경 즉시 `오늘 가능예산`, `저축 예상`을 다시 계산해서 보여준다.
- 목표 chip은 항상 HeroSpendingCard 오른쪽에 유지한다.

### 5. 소비 자동 분류

표시 위치:

- 거래 내역 행의 보조 텍스트
- 선택일 상세 패널
- 코치 화면의 카테고리 코멘트

디자인:

- 카테고리 색을 과하게 많이 쓰지 않는다.
- 홈에서는 `카페 · 오늘 12:41`처럼 텍스트 중심으로 표시한다.
- 캘린더 상태 색은 예산 상태가 우선이고, 카테고리는 상세에서 보여준다.

### 6. 소비 캘린더

표시 위치:

- 홈 중앙
- 캘린더 탭 전체 화면

디자인:

- 홈에서는 한 달 전체보다 "이번 주 흐름"처럼 빠르게 읽히는 요약을 우선한다.
- 캘린더 탭에서는 월 전체를 확장한다.
- 셀 상태는 `safe`, `subscription`, `over`, `selected`를 기본으로 한다.

셀 예시:

```html
<button class="calendar-day is-over">
  <span class="calendar-day-number">14</span>
  <span class="calendar-day-meta">4.2만</span>
</button>
```

### 7. 구독 내역

표시 위치:

- HeroSpendingCard의 `고정 구독비`
- MiniInsightCard의 `구독 지출`
- 캘린더의 구독일 앰버 표시
- 코치 화면의 구독 점검 미션

디자인:

- 구독은 앱 차별점이므로 홈에서 반드시 한 번 이상 보인다.
- 자동 해지를 구현하지 않더라도 `점검`, `알림`, `예상 절약`은 보여줄 수 있다.

카피:

```text
구독 1개 점검하기
어도비 결제 전 알림 예정
```

### 8. AI 소비 코치

표시 위치:

- HeroSpendingCard 안 CoachStrip
- MissionList
- 코치 탭

디자인:

- AI 채팅창으로 시작하지 않는다.
- 홈에서는 한 문장과 미션 2개만 보여준다.
- 자세한 설명은 코치 탭으로 보낸다.

좋은 구조:

```text
행동: 목요일 무지출 하루 만들기
이유: 교통비 제외 자동 계산
예상 절약: +9천
```

### 9. 최근 소비

표시 위치:

- 홈 하단
- 캘린더 선택일 상세

디자인:

- 홈에서는 2건만 보여준다.
- 금액 색은 지출 코랄, 입금 민트로 고정한다.
- 거래처 아이콘은 단순 문자 또는 서비스 약자로 충분하다.

## 화면 확장 기준

### 홈에서 보여줄 것

- 전체 소비 상태
- 오늘의 조정
- 소비 캘린더 요약
- 미션
- 구독/잔액 보조 정보
- 최근 내역

### 캘린더 탭으로 보낼 것

- 월 전체 내역
- 날짜별 상세 거래
- 카테고리 필터
- 과소비일 필터

### 입력 탭으로 보낼 것

- 직접 입력
- CSV 업로드
- 반복 지출 등록

### 코치 탭으로 보낼 것

- 긴 AI 설명
- 미션 전체 목록
- 목표 조정 제안
- 구독 점검 상세

### 설정으로 보낼 것

- 개인정보 안내
- 샘플 데이터 초기화
- 알림 설정
- 목표 기본값

## 구현할 때 피해야 할 흔한 실수

1. 모든 분석 결과를 카드로 만들어 홈에 쌓는다.
2. 캘린더 셀 안에 긴 텍스트를 넣는다.
3. AI를 말풍선 채팅 UI로 만든다.
4. 보라색 그라데이션 배경을 추가한다.
5. 구독 기능을 별도 페이지에 숨긴다.
6. 하단 탭이 최근 내역을 덮도록 둔다.
7. 목표 설정 후 홈의 숫자가 바뀌지 않는다.
8. 입금과 지출을 같은 색으로 보여준다.

## 기능별 완료 기준

기능이 완성되었다고 판단하려면 기능 동작과 함께 디자인 기준도 통과해야 한다.

| 기능 | 디자인 완료 기준 |
| --- | --- |
| 샘플 데이터 | 홈 카드, 캘린더, 최근 내역이 v4 컴포넌트로 채워짐 |
| 목표 설정 | 목표 chip, 진행률, 오늘 가능예산이 즉시 갱신됨 |
| 직접 입력 | 입력 후 선택일 캘린더 상태와 최근 내역이 갱신됨 |
| CSV 연결 | 불러온 건수와 구독 후보가 짧은 피드백으로 표시됨 |
| AI 코치 | 홈에는 한 문장, 미션에는 행동/이유/절약액이 표시됨 |
| 구독 분석 | 홈, 캘린더, 코치 중 최소 2곳에서 연결되어 보임 |
