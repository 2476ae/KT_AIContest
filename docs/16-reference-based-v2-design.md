# 레퍼런스 기반 v2 디자인 방향

## 왜 v2가 필요한가

v1 목업은 정보 구조를 확인하기에는 좋지만, 실제 배포 서비스처럼 보이기에는 부족합니다. 화면이 웹 대시보드처럼 보이고, AI가 만든 MVP 특유의 균일한 카드 배열과 건조한 느낌이 강합니다.

v2의 목표는 다음입니다.

> 머니루틴 캘린더를 "AI 대시보드"가 아니라 실제 모바일 금융앱처럼 보이게 만든다.

## 사용자가 제공한 레퍼런스에서 뽑은 공통 패턴

### 1. 모바일 우선 구조

대부분의 금융앱은 모바일 화면에서 시작합니다. 데스크톱 대시보드가 아니라, 한 손으로 스크롤하는 홈 피드가 중심입니다.

적용:

- 홈 화면은 모바일 기준으로 먼저 디자인합니다.
- 데스크톱은 모바일 홈을 확장한 웹앱 레이아웃으로 봅니다.
- 첫 구현도 모바일 화면 완성도를 먼저 잡습니다.

### 2. 큰 숫자와 짧은 판단

금융앱은 잔액, 소비액, 예정 금액을 굵고 크게 보여줍니다. 설명은 짧고 숫자가 먼저 보입니다.

적용:

- "이번 달 지출", "남은 하루 예산", "구독 고정비"를 크게 보여줍니다.
- AI 리포트는 긴 문단이 아니라 한 줄 판단과 액션으로 줄입니다.

### 3. 홈 피드형 카드 리듬

레퍼런스 화면들은 계좌 카드, 혜택 카드, 배너, 최근 내역, 기능 그리드가 리듬 있게 이어집니다. 모든 카드가 같은 크기와 무게가 아닙니다.

적용:

- 카드 4개를 균등하게 나열하지 않습니다.
- 메인 카드 1개, 보조 카드 2개, 피드 카드 여러 개로 강약을 만듭니다.

### 4. 하단 탭의 앱다운 인상

금융앱은 하단 탭이 강합니다. 홈, 자산/소비, 혜택, 전체 같은 구조가 앱의 완성도를 만듭니다.

적용:

- 모바일 하단 탭은 크게 유지합니다.
- 탭은 `홈`, `캘린더`, `입력`, `AI코치`, `설정` 또는 4개 축약 버전으로 갑니다.

### 5. 캘린더 화면은 더 과감해도 된다

레퍼런스 1처럼 캘린더는 어두운 테마나 강한 숫자 밀도를 써도 잘 어울립니다. 특히 수입/지출을 색으로 나누면 금융앱다운 느낌이 강합니다.

적용:

- 홈은 밝고 부드럽게 갑니다.
- 소비 캘린더 상세 화면은 더 밀도 있고 차분한 다크/딥톤 옵션을 고려합니다.
- 수입은 파랑, 지출은 회색/코랄, 초과는 빨강 계열로 분리합니다.

### 6. 프로모션 대신 AI 코칭 피드

금융앱의 배너/혜택 영역을 우리 앱에서는 AI 코칭 피드로 바꿉니다.

적용:

- "우리청년미래적금 가입" 같은 배너 자리에 "이번 주 카페 2회 줄이면 12,000원 절약"을 넣습니다.
- 광고처럼 보이지 않고, 개인화된 금융 코칭처럼 보여야 합니다.

| --- | --- |

## v2 홈 화면 구조

모바일 기준 순서는 다음입니다.

1. 상단 앱바: 서비스명, 알림, 전체 메뉴
2. AI 코치 말풍선: 오늘의 한 줄 판단
3. 메인 소비 카드: 이번 달 지출, 남은 하루 예산, 목표 상태
4. 빠른 액션: 입력, CSV, 목표, 리포트
5. 미니 소비 캘린더: 이번 주/이번 달 소비 강도
6. AI 미션 피드: 절약 미션 2~3개
7. 구독 점검 카드: 월 고정비와 점검 후보
8. 최근 소비 내역
9. 하단 탭

## v2 화면별 방향

### 홈

기존 대시보드가 아니라 금융앱 홈처럼 갑니다.

- 카드 4개 균등 배치 금지
- 메인 소비 카드 1개를 크게 사용
- AI 코치 말풍선으로 서비스 개성 부여
- 하단에는 최근 내역과 구독 점검을 피드처럼 배치

### 소비 캘린더

레퍼런스 1의 밀도 있는 달력 감각을 참고합니다.

- 날짜 아래에 수입/지출 금액을 작게 표시
- 선택 날짜는 카드처럼 떠 보이게 강조
- 검색/필터를 상단에 둠
- 하단에는 해당 날짜 소비 내역과 AI 코멘트

### 입력

금융앱의 송금/충전 화면처럼 단순하고 빠르게 갑니다.

- 직접 입력은 큰 금액 입력부터 시작
- 사용처, 날짜, 카테고리는 아래 보조 필드
- CSV는 별도 카드로 보조 배치

### AI 리포트

금융앱의 혜택/피드 화면처럼 구성합니다.

- 한 줄 결론
- 이번 주 미션
- 구독 점검
- 카테고리별 분석
- 최근 3개월 비교

## 기존 v1에서 버릴 것

- 균등한 4개 카드가 첫 화면을 지배하는 구조
- 웹 대시보드 같은 상단 탭 중심 구조
- 너무 얇고 건조한 캘린더 셀
- 모든 화면이 같은 흰 카드 반복으로 보이는 구조
- "AI 오늘의 판단"이라는 다소 설명적인 제목

## 새로 가져갈 것

- 앱바와 하단 탭의 모바일 앱다운 구조
- AI 코치 말풍선
- 큰 메인 소비 카드
- 피드형 미션 카드
- 구독 점검 카드
- 최근 소비 내역
- 캘린더 상세의 밀도 있는 숫자 표현
