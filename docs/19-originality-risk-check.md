# 우리은행 유사성 리스크 점검

## 결론

현재 v4 디자인은 전체적으로 우리은행 앱을 직접 복제한 수준은 아니다. 다만 금융앱 레퍼런스를 참고한 만큼, 특정 은행을 떠올리게 하는 직접 요소는 제거해야 한다.

가장 위험했던 요소는 목업의 최근 내역에 들어간 `우리 WON 통장` 텍스트였다. 이는 특정 은행의 상품명/브랜드 표현과 직접 연결될 수 있으므로 `샘플 입금 계좌`로 교체했다.

## 낮은 리스크 요소

- 밝은 회색 배경과 흰 카드
- 큰 금액 카드
- 하단 탭
- 계좌/소비 요약 카드

위 요소는 모바일 금융앱 전반에서 흔한 UX 패턴이다. 특정 앱의 고유 표현을 그대로 쓰지 않는 한, 일반적인 패턴 참고로 볼 수 있다.

## 주의가 필요한 요소

- 특정 은행명 또는 상품명 사용
- `WON`, `우리`, `Woori` 등 직접 브랜드 연상 문구
- 우리은행 로고와 유사한 파란 원형 W 마크
- 우리은행 캐릭터/일러스트와 유사한 배너
- 계좌 카드의 문구, 번호, 버튼 배치를 그대로 재현하는 방식

## 차별화 기준

우리 서비스는 아래 요소로 차별화한다.

1. 첫 화면 중심이 계좌 잔액이 아니라 `소비 캘린더`다.
2. 광고/혜택 배너가 아니라 `오늘의 조정`과 `이번 주 미션`이 보인다.
3. 구독 지출과 목표 저축을 한 화면에서 연결한다.
4. 브랜드명은 `머니루틴`이고, 로고는 `M` 기반의 독립 마크를 사용한다.
5. 색상은 파란색 단독이 아니라 블루, 민트, 코랄, 앰버의 역할 기반 시스템을 사용한다.

## 구현 시 금지

- 샘플 데이터에 실제 은행명, 카드명, 상품명을 넣지 않는다.
- `우리 WON 통장`, `WON뱅킹`, `우리카드` 같은 표현을 쓰지 않는다.
- 은행 앱의 혜택 배너 구조를 그대로 복제하지 않는다.
- 캐릭터형 금융 광고 배너를 넣지 않는다.
- 파란색 원형 W 계열 로고를 만들지 않는다.

## 권장 샘플명

- 샘플 입금 계좌
- 생활비 통장
- 월급 입금
- 체크카드 결제
- 동네 카페
- 교통카드 충전

## 최종 판단

현재 v4 방향은 유지해도 된다. 다만 기능 구현 단계에서는 `머니루틴만의 중심 경험`, 즉 소비 캘린더와 목표 코치가 첫 화면에서 더 강하게 보여야 한다.
