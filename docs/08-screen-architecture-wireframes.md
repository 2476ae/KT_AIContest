# 화면 구조와 와이어프레임

## 앱 구조

머니루틴 캘린더의 화면은 다섯 개로 구성합니다.

1. 대시보드
2. 소비 캘린더
3. 소비 내역
4. 목표 설정
5. AI 리포트

다만 첫 제출작에서는 대시보드 화면 안에서 대부분의 흐름을 경험할 수 있게 설계합니다. 별도 화면 이동이 많으면 3분 데모의 힘이 약해집니다.

## 내비게이션

데스크톱:

- 좌측 사이드 내비게이션 또는 상단 탭 중 하나를 선택합니다.
- 추천은 상단 탭입니다. 공모전 데모에서는 화면 폭을 캘린더에 더 많이 쓰는 편이 좋습니다.

모바일:

- 하단 탭 4개를 사용합니다.
- 홈, 캘린더, 입력, AI 리포트

## 화면별 우선순위

### 대시보드

목적:

- 사용자가 현재 상태를 즉시 이해합니다.

최상단 정보:

- 이번 달 지출
- 목표 소비액 대비 사용률
- 남은 하루 예산
- 목표 저축 가능성

금지:

- 서비스 소개 문구가 화면 대부분을 차지하는 것
- 의미 없는 큰 일러스트

### 소비 캘린더

목적:

- 날짜별 소비 흐름을 직관적으로 보여줍니다.

주요 상태:

- 일반 소비일
- 과소비일
- 무지출일
- 구독 결제일
- 오늘
- 선택한 날짜

날짜 클릭 시:

- 총 지출
- 거래 목록
- 카테고리 비중
- AI 한 줄 코멘트

### 소비 내역

목적:

- 거래를 입력하고 분류를 수정합니다.

주요 요소:

- 직접 입력 버튼
- CSV 업로드 버튼
- 카테고리 필터
- 거래 목록
- 카테고리 수정 메뉴

### 목표 설정

목적:

- AI 분석 기준을 조정합니다.

주요 요소:

- 월 수입
- 목표 소비액
- 목표 저축액
- 카테고리별 예산
- 우선 절약 항목

입력 UX:

- 숫자 입력에는 원 단위 포맷을 적용합니다.
- 추천값 버튼을 제공합니다.

### AI 리포트

목적:

- 분석 결과를 행동으로 바꿉니다.

구성:

- 한 줄 결론
- 상태 라벨
- 남은 하루 예산
- 소비 인사이트 3개
- 절약 미션 3개
- 구독 점검 조언

## 정보 밀도 규칙

- 데스크톱은 캘린더와 AI 판단을 같은 높이에 배치합니다.
- 모바일은 AI 판단을 캘린더보다 위에 둡니다. 작은 화면에서는 결론을 먼저 보여주는 편이 좋습니다.
- 한 화면에 금액 카드가 4개를 넘지 않게 합니다.
- 긴 문단보다 한 줄 결론과 짧은 리스트를 사용합니다.
