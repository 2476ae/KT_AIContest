# AI 기능 설계와 프롬프트

## AI의 역할

머니루틴 캘린더에서 생성형 AI는 챗봇 장식이 아니라 소비 의사결정을 돕는 보조 도구입니다. 다만 금융 숫자까지 AI에 맡기지는 않습니다. 현재 배포 버전은 검증 가능한 로컬 계산과 생성형 AI의 장점을 다음처럼 나누어 사용합니다.

1. 사용자가 자동 분류를 선택한 거래는 OpenAI가 소비 카테고리를 분류합니다.
2. 구독·반복 결제 후보는 거래 주기와 사용처를 기준으로 로컬 규칙이 탐지합니다.
3. 목표 소비액, 목표 저축액, 오늘 한도와 저축 전망은 TypeScript 로직이 계산합니다.
4. OpenAI는 계산이 끝난 카테고리 행동과 미션 문구를 더 자연스럽게 다듬습니다.

이 경계를 둔 이유는 생성형 AI의 문장 능력을 활용하면서도 예산 숫자와 상태가 임의로 바뀌는 일을 막기 위해서입니다.

## AI 출력 원칙

- 사용자를 비난하지 않습니다.
- 어려운 금융 용어를 쉬운 말로 바꿉니다.
- 소비를 무조건 줄이라고 하지 않고, 목표 달성을 위한 선택지를 제시합니다.
- 대출, 투자, 보험 등 고위험 금융상품을 추천하지 않습니다.
- 개인정보와 계좌 인증정보 입력을 요구하지 않습니다.

## 카테고리 분류 체계

기본 카테고리는 다음으로 시작합니다.

- 식비
- 카페/간식
- 교통
- 쇼핑
- 여가
- 구독
- 교육
- 의료
- 생활
- 기타

사용자가 직접 카테고리를 수정할 수 있고, 수정된 분류는 이후 추천 기준에 반영하는 UX를 제공합니다.

## 프롬프트 1: 소비 카테고리 분류

```text
너는 청년 소비 관리를 돕는 금융 코칭 AI다.
아래 거래내역을 보고 가장 적절한 소비 카테고리를 하나 선택하라.

카테고리 목록:
식비, 카페/간식, 교통, 쇼핑, 여가, 구독, 교육, 의료, 생활, 기타

판단 기준:
- 반복 결제형 서비스명이나 월 정액 서비스는 구독으로 분류한다.
- 식당, 편의점 도시락, 배달앱은 식비로 분류한다.
- 커피, 베이커리, 디저트는 카페/간식으로 분류한다.
- 지하철, 버스, 택시, 공유자전거는 교통으로 분류한다.
- 분류가 애매하면 기타로 두고 이유를 짧게 설명한다.

거래내역:
거래일: {{date}}
사용처: {{merchant}}
금액: {{amount}}원
메모: {{memo}}

출력은 JSON으로만 작성하라.
{
  "category": "카테고리명",
  "confidence": 0.0,
  "reason": "짧은 이유"
}
```

## 로컬 규칙 2: 구독/반복 결제 탐지

```text
입력: 사용처, 금액, 날짜, 사용자가 표시한 정기결제 여부

1. 사용자가 정기결제로 표시한 거래를 우선 후보로 본다.
2. 같은 사용처에서 비슷한 금액이 반복되는지 확인한다.
3. 월 예상 금액과 다음 결제일을 로컬에서 계산한다.
4. 사용 빈도와 설정한 구독 상한을 비교해 유지 또는 점검 후보로 표시한다.
```

이 과정은 전체 거래내역을 외부 AI로 보내지 않도록 현재 버전에서 로컬 로직으로 처리합니다.

## 프롬프트 3: 목표 기반 소비 코칭 문구 보완

```text
모든 응답은 자연스러운 한국어 완결 문장으로 작성한다.
너는 대학생과 사회초년생을 위한 목표 기반 소비 코치다.
payload.baseReport는 앱이 검증한 최종 계산 결과다.
숫자, 상태, 판단 방향을 바꾸거나 반박하지 않는다.
카테고리 행동과 미션의 제목·행동 문구만 더 친절하고 구체적으로 다듬는다.
금액이 계획 안이면 초과, 줄이기, 절감 필요라고 쓰지 않는다.
사용자를 탓하지 않고 각 문장을 짧게 쓴다.
과장된 금융 조언, 투자 권유, 계좌 연결 요청은 하지 않는다.
```

## AI 활용 설명 문장

지원서에는 다음 방향으로 작성합니다.

머니루틴 캘린더는 생성형 AI를 소비 내역 분류와 목표 기반 행동 문구 보완에 활용합니다. 사용자가 자동 분류를 선택하면 AI가 사용처명과 메모를 바탕으로 소비 유형을 정합니다. 예산 진행률, 오늘 한도, 저축 전망과 반복 결제 후보는 로컬 규칙으로 먼저 계산하며 AI가 해당 숫자나 상태를 변경할 수 없습니다. 금융 인증정보를 요구하지 않고 샘플 데이터, 직접 입력, CSV 업로드를 활용해 개인정보 수집을 최소화합니다.
