# 캘린더, AI UX, 비주얼 시스템

## 소비 캘린더 디자인 규칙

캘린더는 머니루틴 캘린더의 핵심 차별점입니다. 단순 날짜판이 아니라 소비 흐름을 읽는 지도처럼 보여야 합니다.

## 날짜 셀 상태

| 상태 | 표현 | 의미 |
| --- | --- | --- |
| 기본일 | 흰 배경, 얇은 테두리 | 거래가 없거나 일반적인 날 |
| 소비일 | 옅은 카테고리 점 또는 금액 표시 | 소비가 발생한 날 |
| 과소비일 | 코랄 테두리 + "초과" 라벨 | 일평균 권장 예산을 넘은 날 |
| 무지출일 | 민트 배경 + "0원" 라벨 | 지출이 없는 날 |
| 구독일 | 작은 고정비 뱃지 | 반복 결제 또는 구독 결제일 |
| 오늘 | 진한 잉크색 외곽선 | 현재 날짜 |
| 선택일 | 강조 배경 + 상세 패널 연결 | 사용자가 클릭한 날짜 |

## 날짜 셀 내부 정보

데스크톱:

- 날짜 숫자
- 하루 총 지출
- 최대 2개의 카테고리 점
- 구독 뱃지
- 상태 라벨

모바일:

- 날짜 숫자
- 소비 강도 점
- 구독 뱃지
- 선택 시 하단 패널에서 상세 표시

## 소비 강도 단계

하루 지출액을 목표 하루 예산과 비교해 4단계로 표시합니다.

| 단계 | 기준 | 표현 |
| --- | --- | --- |
| 낮음 | 하루 예산의 50% 이하 | 옅은 민트 |
| 보통 | 하루 예산의 50~100% | 옅은 블루 |
| 주의 | 하루 예산의 100~150% | 옅은 앰버 |
| 위험 | 하루 예산의 150% 초과 | 옅은 코랄 |

색상만으로 판단하지 않도록 `안정`, `주의`, `초과` 같은 텍스트 라벨을 함께 사용합니다.

| 카테고리 | 색상 | 용도 |

| 교통 | #3C78B5 | 지하철, 버스, 택시, 공유자전거 |
| 쇼핑 | #7C5AA6 | 의류, 잡화, 온라인 쇼핑 |
| 여가 | #2C9A8A | 영화, 모임, 취미 |
| 구독 | #2F3645 | OTT, 음악, 앱, 멤버십 |
| 교육 | #5A8F3D | 책, 강의, 학습 |
| 의료 | #C65C7B | 약국, 병원 |

카테고리 색상은 전체 배경색으로 크게 쓰지 않고, 점, 뱃지, 얇은 바, 작은 그래프에 사용합니다.

## 비주얼 팔레트

| 역할 | 색상 | 설명 |
| --- | --- | --- |
| 기본 배경 | #F6F8F5 | 차갑지 않은 밝은 배경 |
| 표면 | #FFFFFF | 카드와 패널 |
| 기본 텍스트 | #151A17 | 높은 가독성 |
| 보조 텍스트 | #68736D | 설명과 라벨 |
| 라인 | #DDE4DF | 경계선 |
| 주 색상 | #2F7D5C | 신뢰감 있는 녹색 |
| 보조 색상 | #315D86 | 금융 신뢰감 |
| 주의 | #D99021 | 예산 주의 |
| 위험 | #D95A45 | 초과 경고 |
| 성공 | #4A9C6E | 안정, 무지출 |

주의:

- 보라색/블루 그라디언트가 화면을 지배하지 않게 합니다.
- 배경에 장식용 구체, 블러, 큰 그라디언트 장식을 사용하지 않습니다.
- 금융 앱답게 대비와 숫자 가독성을 우선합니다.

## 타이포그래피

권장:

- 한국어 UI는 `Pretendard`, `Noto Sans KR`, 시스템 산세리프 순서로 사용합니다.
- 숫자는 같은 폭으로 보이게 `tabular-nums`를 적용합니다.

크기 기준:

| 용도 | 크기 |
| --- | ---: |
| 페이지 제목 | 28~32px |
| 섹션 제목 | 18~22px |
| 카드 숫자 | 24~32px |
| 본문 | 15~16px |
| 보조 라벨 | 12~13px |

규칙:

- 글자 간격은 0으로 둡니다.
- 버튼 안의 문장은 한 줄에 무리하게 우겨 넣지 않습니다.
- 금액은 쉼표와 원 단위를 일관되게 표시합니다.

## 형태와 간격

- 카드 radius는 8px 이하를 기본으로 합니다.
- 캘린더 셀은 고정 높이를 갖고, 상태 변화로 크기가 흔들리지 않게 합니다.
- 버튼과 탭은 최소 40px 높이를 유지합니다.
- 섹션 간 간격은 24~32px, 카드 내부 여백은 16~20px를 기본으로 합니다.

## AI UX 규칙

AI는 "대화창"보다 "판단 카드"와 "미션 카드"로 보여줍니다.

### AI 판단 카드 구조

1. 상태 라벨: 안정 / 주의 / 위험
2. 한 줄 결론
3. 핵심 숫자
4. 이유 2~3개
5. 바로 할 행동 1개

예시:

```text
주의
남은 12일 동안 하루 13,500원 이하로 쓰면 목표 저축액에 도달할 수 있어요.

이유
- 카페/간식 지출이 목표보다 빠르게 늘고 있어요.
- 주말 배달 지출이 반복되고 있어요.

오늘 할 일
이번 주 카페 구매를 2회만 줄여보세요.
```

### 미션 카드 구조

- 미션명
- 예상 절약액
- 난이도
- 실천 방법
- 완료 체크

예시:

```text
카페 2회 줄이기
예상 절약 12,000원 / 난이도 쉬움
이번 주 평일 커피 구매를 3회에서 1회로 줄여보세요.
```

## AI 문장 톤

좋은 톤:

- "줄이면 좋아요"
- "도달할 수 있어요"
- "점검해보세요"
- "가장 효과가 큰 항목은..."

피할 톤:

- "과소비했습니다"
- "잘못된 소비입니다"
- "반드시 줄이세요"
- "투자 상품을 추천합니다"

## 개인정보 신뢰 UX

첫 화면 또는 데이터 입력 영역에 다음 문장을 짧게 노출합니다.

> 이 데모는 금융 인증정보를 받지 않습니다. 샘플 데이터, 직접 입력, CSV 업로드로만 분석합니다.

문구는 작지만 눈에 들어오게 배치합니다. 보안 안내가 너무 커지면 서비스가 불안해 보일 수 있으므로 보조 정보로 처리합니다.
