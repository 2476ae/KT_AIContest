# AI and UX Improvement Backlog

## 처리 완료

1. 알림 읽음 처리
   - 사용자가 알림 패널을 열면 현재 알림 ID를 로컬 저장소에 읽음 처리한다.
   - 상단 벨 badge count는 전체 알림 수가 아니라 미확인 알림 수만 표시한다.

2. AI 역할 확장
   - 한 번의 `OpenAI 분석 업데이트` 호출 안에서 소비 패턴 진단, 분야별 소비 계획, 현실 조정 플랜, 절약 미션, 구독 점검을 함께 생성하도록 `CoachReport` 구조를 확장했다.
   - `categoryPlans`를 추가해 식비, 카페/간식, 구독 등 분야별 현재 금액, 계획 금액, 예상 절약액, 실행 행동을 카드로 보여준다.

3. 긴 문장 UI 방지
   - OpenAI provider 검증 단계에서 headline, todayAction, insights, missions, categoryPlans, basis 문구를 짧게 제한한다.
   - 화면에는 짧은 라벨, 이유 1줄, 행동 1개, 예상 절약액만 표시한다.

4. API 사용량 방어선
   - AI 코치는 명시 버튼 클릭에서만 호출하고 같은 입력값은 캐시한다.
   - 브라우저 로컬 일일 제한을 추가했다. 기본값은 전체 8회, AI 코치 5회, 자동 분류 8회다.
   - 서버 프록시에 출력 토큰 상한과 서버 일일 제한을 추가했다.

5. AI 코치 화면 과밀도 완화
   - 분야별 소비 계획 카드는 최대 4개로 제한하고, 안정적인 항목은 과하게 노출하지 않는다.
   - 중복 정보였던 분야별 지출 흐름 바 섹션은 AI 코치 화면에서 제거했다.
   - 정기 결제는 구독 상한뿐 아니라 전체 예산 대비 비중도 함께 보고, 부담이 낮으면 점검 미션을 띄우지 않는다.

## 남은 제출 전 확인

1. Vercel 최신 배포 확인
   - `https://kt-ai-contest.vercel.app/`에 최신 커밋이 반영됐는지 확인한다.
   - OpenAI 버튼은 비용을 고려해 필요한 경우만 누른다.

2. 실제 브라우저 QA
   - 알림 읽음 처리, 분야별 소비 계획 카드, AI 코치 로딩 오버레이, 목표 초과/미초과 상태를 PC와 모바일에서 확인한다.

3. 예약 QA 토큰 관리
   - 예약 작업의 앱 내부 AI 호출은 테스트 사이클당 최대 5회로 유지한다.
   - 테스트 결과에는 실제 호출 횟수를 기록한다.

## 이후 AI 기능 후보

1. 소비 패턴 진단
   - 이번 달 소비 구조를 짧은 원인 카드로 정리한다.
   - 예: `월말 소비 증가`, `구독 상한 근접`, `쇼핑 비중 높음`.

2. 현실 조정 플랜
   - 월 목표 초과 시 월수입, 저축 목표, 남은 일수를 고려해 현실 조정 목표를 설명한다.
   - 금액 계산은 로컬 로직이 최종 기준이고 AI는 이유와 실행안을 보조한다.

3. 구독 위험도 판단
   - 반복 결제를 `유지`, `점검`, `해지 검토`로 구분하고 짧은 이유를 제공한다.
   - 예: `월 고정비 비중 높음`, `목표 상한 근접`.

4. 이상 소비 알림
   - 새 소비가 들어왔을 때 목표 초과 위험, 카테고리 급증, 정기 결제 반영 등을 알림으로 만든다.

## 화면 원칙

- AI는 챗봇처럼 긴 설명을 하지 않는다.
- 앱 화면에는 카드형 판단과 실행 항목만 둔다.
- 금액, 목표 초과 여부, 하루 한도는 로컬 계산값을 최종 기준으로 한다.
- OpenAI 호출은 사용자 명시 행동에만 연결한다.
- 같은 입력값은 캐시를 재사용한다.
