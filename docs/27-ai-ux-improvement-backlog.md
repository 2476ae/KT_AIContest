# AI and UX Improvement Backlog

## 우선 보완

1. 알림 읽음 처리
   - 알림 패널을 열어 확인했는데도 상단 벨 버튼 숫자가 사라지지 않는다.
   - 알림별 `read` 상태 또는 마지막 확인 시각을 저장해야 한다.
   - 사용자가 알림 패널을 열면 현재 알림을 읽음 처리하고 badge count는 미확인 알림 수만 표시한다.

2. AI 역할 확장
   - 현재 AI는 자동 분류와 코치 문장 보조 역할이 강해 핵심 기능처럼 보이기 어렵다.
   - 한 번의 `OpenAI 분석 업데이트` 호출 안에서 소비 패턴 진단, 분야별 소비 계획, 현실 조정 플랜, 절약 미션, 구독 점검을 함께 생성한다.
   - API 호출 횟수는 늘리지 않고 응답 구조를 확장한다.

3. 긴 문장 UI 방지
   - AI 결과를 긴 문단으로 노출하면 앱 화면이 난잡해진다.
   - 화면에는 짧은 라벨, 이유 1줄, 행동 1개, 예상 절약액만 표시한다.
   - 긴 설명은 접힘 영역이나 자세히 보기 안에만 둔다.

## AI 기능 후보

1. 분야별 소비 계획
   - 식비, 카페/간식, 교통, 쇼핑, 여가, 구독, 생활 등 카테고리별 남은 기간 계획을 제안한다.
   - 예: `식비 집중`, `배달 1회 줄이기`, `목표 절약 15,000원`.

2. 소비 패턴 진단
   - 이번 달 소비 구조를 짧은 원인 카드로 정리한다.
   - 예: `월말 소비 증가`, `구독 상한 근접`, `쇼핑 비중 높음`.

3. 현실 조정 플랜
   - 월 목표 초과 시 월수입, 저축 목표, 남은 일수를 고려해 현실 조정 목표를 설명한다.
   - 금액 계산은 로컬 로직이 최종 기준이고 AI는 이유와 실행안을 보조한다.

4. 구독 위험도 판단
   - 반복 결제를 `유지`, `점검`, `해지 검토`로 구분하고 짧은 이유를 제공한다.
   - 예: `월 고정비 비중 높음`, `목표 상한 근접`.

5. 이상 소비 알림
   - 새 소비가 들어왔을 때 목표 초과 위험, 카테고리 급증, 정기 결제 반영 등을 알림으로 만든다.

## 화면 원칙

- AI는 챗봇처럼 긴 설명을 하지 않는다.
- 앱 화면에는 카드형 판단과 실행 항목만 둔다.
- 금액, 목표 초과 여부, 하루 한도는 로컬 계산값을 최종 기준으로 한다.
- OpenAI 호출은 사용자 명시 행동에만 연결한다.
- 같은 입력값은 캐시를 재사용한다.
