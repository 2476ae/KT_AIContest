# v4 디자인 토큰과 컴포넌트 시스템

## 목적

이 문서는 기능 구현 단계에서 화면이 v4 구상도의 분위기와 품질을 유지하도록 돕는 디자인 기준이다. 구현 채팅에서는 이전 MVP 목업보다 이 문서를 우선 참고하되, v4 스크린샷을 픽셀 단위로 그대로 재현하지 않는다.

기준 방향은 **밝은 개인 금융 코치형 소비 캘린더**다. 금융앱처럼 신뢰감 있고, 캘린더앱처럼 읽기 쉬우며, AI 서비스처럼 과하게 설명하기보다 개인화된 조정 행동을 제안해야 한다.

구현용 CSS 기준 파일:

- `design/v4-ui-system.css`

대표 시안:

- `mockups/mobile-v4-reference-synthesis.html`
- `mockups/mobile-v4-reference-synthesis.png`

해석:

- 위 시안은 지향점이다.
- 실제 구현에서는 기능, 데이터, 반응형 조건에 맞게 화면을 조정한다.
- 반드시 유지할 것은 색상 역할, 정보 위계, 컴포넌트 리듬, 코치형 카피다.

## 핵심 디자인 원칙

1. 첫 화면은 랜딩페이지가 아니라 실제 앱 홈이어야 한다.
2. "AI 리포트"보다 "오늘의 조정", "이번 주 미션"처럼 행동 중심 카피를 쓴다.
3. 전체 화면은 밝은 톤으로 통일한다. 캘린더만 다크모드로 분리하지 않는다.
4. 큰 숫자, 짧은 문장, 고정된 카드 리듬을 유지한다.
5. 카드 안에 카드를 넣지 않는다. 카드가 필요한 곳은 요약, 캘린더, 미션, 내역처럼 기능 단위다.
6. 장식용 그라데이션, 보라색 AI SaaS 톤, 광고 배너형 캐릭터는 쓰지 않는다.

## 색상 사용 규칙

| 역할 | 색상 | 사용 위치 |
| --- | --- | --- |
| 배경 | `#F5F7FA` | 앱 전체 배경 |
| 표면 | `#FFFFFF` | 카드, 하단 탭, 액션 버튼 |
| 부드러운 표면 | `#F9FBFD` | 카드 내부 보조 영역 |
| 본문 | `#16191F` | 주요 숫자, 제목 |
| 보조 텍스트 | `#697281` | 날짜, 설명, 보조 라벨 |
| 메인 블루 | `#2478FF` | 선택 날짜, 주요 액션, 활성 탭 |
| 소프트 블루 | `#EAF2FF` | 코치 카드, 활성 배경 |
| 세이빙 민트 | `#00A878` | 절약, 무지출, 목표 개선 |
| 경고 코랄 | `#FF665A` | 초과 소비, 위험 신호 |
| 구독 앰버 | `#F6A623` | 구독 결제, 고정비 |

주의:

- 파란색은 주 색이지만 화면 전체를 파랗게 덮지 않는다.
- 위험 상태는 코랄만 사용하지 말고 텍스트 라벨을 함께 둔다.
- 카테고리별 색상은 캘린더 상태 색보다 우선하지 않는다.

## 타이포그래피

### 폰트

```css
font-family:
  Pretendard,
  "Noto Sans KR",
  -apple-system,
  BlinkMacSystemFont,
  "Segoe UI",
  sans-serif;
```

### 크기

| 역할 | 모바일 기준 |
| --- | ---: |
| 앱 타이틀 | 22px |
| 메인 금액 | 30px |
| 섹션 제목 | 17~18px |
| 카드 핵심 숫자 | 15~21px |
| 미션/내역 제목 | 13~14px |
| 보조 설명 | 11~12px |

규칙:

- `letter-spacing`은 0으로 둔다.
- 금액에는 `font-variant-numeric: tabular-nums`를 적용한다.
- 카드 내부 설명은 가능하면 2줄 안에 끝낸다.
- 히어로 크기의 글씨를 작은 카드 안에서 반복하지 않는다.

## 간격과 형태

| 용도 | 값 |
| --- | ---: |
| 모바일 화면 최대 폭 | 430px |
| 앱 바깥 여백 | 18px |
| 작은 화면 바깥 여백 | 14px |
| 섹션 제목 위 간격 | 18px |
| 카드 내부 여백 | 12~18px |
| 카드/버튼 radius | 8px |
| 하단 탭 확보 여백 | 82px 이상 |

규칙:

- 모든 주요 카드는 8px radius를 기본으로 한다.
- 카드 사이 간격은 8~12px을 기본으로 하고, 섹션 간격만 18px 이상 둔다.
- 하단 고정 탭이 콘텐츠를 가리지 않도록 페이지 하단 패딩을 반드시 둔다.

## 화면 골격

### MobileAppShell

역할:

- 모바일 금융앱 같은 첫인상을 만든다.
- 폭은 430px 기준으로 잡고 중앙 정렬한다.
- 하단 탭을 위한 패딩을 포함한다.

필수 클래스:

- `.app-phone`
- `.app-page`
- `.bottom-nav`

### TopBar

구성:

- 브랜드 마크
- 서비스명
- 짧은 보조 문구
- 월 선택
- 알림

주의:

- 상단 액션은 2개 이하로 둔다.
- 검색, 설정, 알림, 메뉴를 모두 한 줄에 넣지 않는다.

### HeroSpendingCard

구성:

- 기준 날짜와 남은 일수
- 이번 달 소비 금액
- 월 목표 chip
- 소비 진행률 bar
- 오늘 가능예산, 저축 예상, 고정 구독비
- 오늘의 조정 코치 문장

규칙:

- 첫 화면에서 가장 큰 카드는 이 카드다.
- "AI가 분석했어요"보다 "오늘의 조정"으로 표현한다.
- 목표 진행률은 숫자와 bar를 함께 표시한다.

### QuickActionGrid

구성:

- 내역 추가
- 파일 연결
- 목표 수정
- 코치 보기

규칙:

- 4개를 넘기지 않는다.
- 각 액션은 아이콘 1개와 라벨 1개만 둔다.
- 버튼 높이는 74px 안팎으로 통일한다.

### SpendingCalendar

구성:

- 섹션 제목
- 캘린더 카드
- 월/주 흐름 제목
- 안정, 구독, 초과 범례
- 7열 날짜 그리드
- 선택일 상세 패널

날짜 셀 상태:

| 상태 | 클래스 | 의미 |
| --- | --- | --- |
| 기본 | `.calendar-day` | 일반 소비 |
| 안정 | `.is-safe` | 무지출/낮은 소비 |
| 구독 | `.is-subscription` | 구독 또는 고정비 결제 |
| 초과 | `.is-over` | 하루 예산 초과 |
| 선택 | `.is-selected` | 현재 선택한 날짜 |
| 이전/다음 달 | `.is-muted` | 비활성 날짜 |

규칙:

- 날짜 셀 크기는 상태 변화로 흔들리지 않아야 한다.
- 금액은 `4.2만`, `0원`, `넷플`처럼 짧게 표시한다.
- 상세 설명은 셀 안이 아니라 선택일 패널에 둔다.

### MissionCard

구성:

- 아이콘
- 행동 미션
- 보조 설명
- 예상 절약액

규칙:

- 미션은 사용자가 바로 실행할 수 있는 행동이어야 한다.
- "절약하세요" 대신 "목요일 무지출 하루 만들기"처럼 구체적으로 쓴다.
- 금액은 오른쪽에 작지만 선명하게 표시한다.

### MiniInsightCard

구성:

- 구독 지출
- 잔액 흐름
- 카테고리 경고
- 목표 상태

규칙:

- 모바일 홈에서는 2열 카드로 배치한다.
- 카드 하나에는 결론 하나만 넣는다.

### TransactionList

구성:

- 거래처
- 카테고리/시간
- 금액
- 입출금 색상

규칙:

- 최근 내역은 기능 신뢰감을 주는 보조 요소다.
- 홈 첫 화면에서는 2~3건만 보여주고 전체 보기는 캘린더/내역 화면으로 넘긴다.

### BottomNavigation

탭:

- 홈
- 캘린더
- 추가
- 코치
- 설정

규칙:

- 모바일에서는 항상 고정한다.
- 가운데 추가 버튼만 강하게 강조한다.
- 활성 탭은 파란 아이콘 배경으로 표시한다.

## 기능 구현 시 상태 매핑

### BudgetStatus

| 상태 | UI 표현 |
| --- | --- |
| `stable` | 민트 보조 배경, "안정" |
| `watch` | 앰버 보조 배경, "점검" |
| `over` | 코랄 보조 배경, "초과" |

### DayStatus

| 상태 | UI 표현 |
| --- | --- |
| `empty` | 기본 회색 셀 |
| `normal` | 기본 회색 셀 + 금액 |
| `safe` | 민트 셀 + 점 |
| `subscription` | 앰버 셀 + 점 |
| `over` | 코랄 셀 + 점 |
| `selected` | 파란 셀 + 흰 텍스트 |

### TransactionType

| 상태 | UI 표현 |
| --- | --- |
| `expense` | 금액 코랄 |
| `income` | 금액 민트 |
| `transfer` | 본문색 또는 보조색 |

## 카피 기준

좋은 카피:

- `카페를 이번 주 2회만 유지하면 목표 저축액에 12,000원 더 가까워져요.`
- `14일은 평소보다 카페 지출이 큰 날`
- `구독 1개 점검하기`
- `이번 달 고정비의 58%가 25일 전 결제돼요.`

피해야 할 카피:

- `AI 리포트 결과`
- `사용자의 소비 패턴을 분석했습니다`
- `절약이 필요합니다`
- `위험합니다`

## 구현 체크리스트

- 첫 화면 상단 50% 안에 서비스명, 이번 달 소비 금액, 목표 진행률이 보이는가?
- 캘린더가 밝은 카드 안에 있으며 앱 전체와 같은 톤인가?
- 주요 액션이 4개 이하인가?
- AI 기능이 긴 설명문이 아니라 조정 문장과 미션으로 보이는가?
- 하단 탭이 콘텐츠를 가리지 않는가?
- 360px 폭에서도 금액과 버튼 텍스트가 넘치지 않는가?
- 보라/파랑 그라데이션, 장식용 배경, 광고 배너가 없는가?
