# 개발 로드맵

## 목표

2026년 7월 15일 17:00 제출 전까지, 심사위원이 URL에 접속해 전체 흐름을 직접 체험할 수 있는 웹앱을 완성합니다.

## 우선순위

### P0: 반드시 완성

- 샘플 데이터 불러오기
- 소비 내역 직접 입력
- 소비 카테고리 자동 분류
- 월간 소비 캘린더
- 목표 소비액/저축액 설정
- AI 절약 리포트
- 개인정보 미수집 안내
- 배포 URL 안정화

### P1: 가능하면 완성

- CSV 업로드
- 카테고리 수동 수정
- 구독 후보 자동 탐지
- 과소비일/무지출일 강조
- 목표 변경 시 리포트 즉시 재계산

### P2: 고도화 계획으로 제시

- 실제 은행/카드 연동
- 오픈뱅킹/마이데이터 연동
- 사용자 계정
- 알림 기능
- 장기 소비 성향 분석

## 구현 순서

### 1일차: 프로젝트 골격

- 프론트엔드 프로젝트 생성
- 기본 라우팅 구성
- 샘플 데이터 로딩
- 공통 레이아웃과 디자인 토큰 구성

### 2일차: 대시보드

- 이번 달 총 지출
- 목표 대비 사용률
- 목표 저축 달성 가능성
- 남은 하루 예산 카드
- AI 한 줄 코멘트 영역

### 3일차: 소비 캘린더

- 월간 캘린더 렌더링
- 날짜별 지출 합계 표시
- 날짜 클릭 시 거래 목록 표시
- 과소비일/무지출일 상태 표시

### 4일차: 소비 입력

- 직접 입력 폼
- 거래 목록
- 카테고리 필터
- 거래 수정/삭제

### 5일차: AI 분류

- 규칙 기반 1차 분류 구현
- AI API 연결 또는 AI 호출 대체 구조 준비
- 분류 이유 표시
- 사용자 수정 UI

### 6일차: 목표 설정

- 월 수입
- 목표 소비액
- 목표 저축액
- 카테고리별 예산
- 우선 절약 항목

### 7일차: AI 리포트

- 카테고리별 지출 분석
- 구독 후보 분석
- 남은 기간 하루 예산 계산
- 절약 미션 3개 생성

### 8일차: CSV 업로드와 데이터 정리

- CSV 파싱
- 필드 검증
- 업로드 결과 미리보기
- 오류 메시지

### 9일차: UX polish

- 모바일 반응형
- 빈 상태
- 로딩 상태
- 오류 상태
- 접근성 라벨

### 10일차: 제출 안정화

- 빌드 오류 제거
- 배포
- 실제 URL 접속 테스트
- 지원서 문장 정리

## 데모 시나리오

최종 제출 전에는 아래 순서가 반드시 끊기지 않아야 합니다.

1. 첫 화면에서 서비스 목적을 이해합니다.
2. 샘플 데이터를 불러옵니다.
3. 목표 소비액과 목표 저축액을 입력합니다.
4. 대시보드에서 현재 상태를 확인합니다.
5. 소비 캘린더에서 과소비일과 구독 결제일을 확인합니다.
6. AI 리포트에서 절약 전략과 다음 주 미션을 확인합니다.

## 검증 체크리스트

- 샘플 데이터 버튼이 항상 정상 작동합니다.
- 새로고침해도 주요 데이터가 유지됩니다.
- 모바일 화면에서 버튼과 텍스트가 겹치지 않습니다.
- 목표 금액을 바꾸면 숫자와 리포트가 함께 바뀝니다.
- AI 분석 결과가 비어 있지 않습니다.
- 금융 인증정보를 입력하라는 흐름이 없습니다.
- 제출 URL이 시크릿 창에서도 열립니다.
